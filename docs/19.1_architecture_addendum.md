## PHẦN 1: DISEASE GRAPH — Tìm Bệnh Nền và Biến Chứng

### 1.1 Vấn đề

Hiện tại chain reasoning là:
```
symptom → pathogenesis → pattern → pháp trị
```

Chain này chỉ trả lời: "Chứng này là gì?" Nó **không** trả lời: "Bệnh nền gây ra triệu chứng này là gì?"

Ví dụ thực tế:
```
User: "già, bị loét thịt do nằm lâu"

Hệ thống hiện tại: → Can khí hư? → Tỳ hư? (chỉ YHCT reasoning)

Hệ thống cần thêm:
  → Pressure ulcer → root cause: immobility
  → Immobility → caused by: stroke? dementia? fracture?
  → Đây là bệnh nền thực sự cần tìm
```

### 1.2 Schema Disease Graph

```sql

CREATE TABLE disease_nodes (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    disease_code    TEXT UNIQUE NOT NULL,
    name_vi         TEXT NOT NULL,
    name_en         TEXT,

-- Edges: quan hệ giữa các bệnh
CREATE TABLE disease_edges (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    source_disease  TEXT REFERENCES disease_nodes(disease_code),
    target_disease  TEXT REFERENCES disease_nodes(disease_code),
    relation_type   TEXT NOT NULL,
    -- relation_type values:
    --   risk_factor    : source là yếu tố nguy cơ của target

    symptom_code    TEXT,
```

## PHẦN 2: HYPOTHESIS ENGINE — Top N Chẩn Đoán

### 2.1 Vấn đề

Hệ thống hiện tại trả về "Chứng tốt nhất" và xếp hạng. Bác sĩ giỏi luôn maintain **danh sách giả thuyết song song** và hỏi câu hỏi để phân biệt chúng.

### 2.2 Cách implement (MVP — không cần engine mới)

HypothesisEngine **không cần** viết thêm class phức tạp. Sử dụng kết quả sẵn có từ YHCTEngine:

```php
class HypothesisEngine {
    /**
     * Lấy top N candidates cạnh tranh nhau từ kết quả engine
     * "Cạnh tranh" = score cách nhau < 20%
     */
    public function getCompetingHypotheses(array $engineResult, int $topN = 3): array {
        $patterns = $engineResult['chung_ranked'];
        if (empty($patterns)) return [];

        $topScore = $patterns[0]['score'];
        $threshold = $topScore * 0.80;  // lấy tất cả trong 20% của top

        return array_filter($patterns, fn($p) => $p['score'] >= $threshold);
    }

    /**
     * Tìm symptom nào phân biệt tốt nhất giữa các candidates
     * Dùng K03.differentiates_from và K04.differentiating_positive/negative
     */
    public function getDifferentiatingSymptoms(array $candidates, array $k03Rules): array {
        $differentiators = [];

        foreach ($candidates as $c1) {
            foreach ($candidates as $c2) {
                if ($c1 === $c2) continue;
                // Tìm symptoms có mặt trong c1 nhưng không có trong c2
                $diff = $this->getDiff($c1, $c2, $k03Rules);
                foreach ($diff as $sym => $weight) {
                    $differentiators[$sym] = ($differentiators[$sym] ?? 0) + $weight;
                }
            }
        }

        arsort($differentiators);
        return array_slice($differentiators, 0, 5, true);
    }
}
```

## PHẦN 3: INFORMATION GAIN QUESTIONING

### 3.1 Vấn đề

Hiện tại re-ranking chỉ đẩy co-occurring symptoms lên đầu. Nhưng bác sĩ giỏi hỏi câu **giúp thu hẹp hypothesis space nhanh nhất**.

### 3.2 Symptom Entropy Score

```php
/**
 * Tính Information Gain của một symptom:
 * "Nếu hỏi symptom này và biết Y/N, giảm được bao nhiêu uncertainty?"
 *
 * Đơn giản hóa cho MVP: symptom có entropy cao = xuất hiện trong nhiều hypothesis
 * nhưng chỉ có mặt trong PHẦN DƯƠNG của một số, PHẦN ÂM của số khác
 */
function calculateEntropyScore(string $symptomCode, array $hypotheses, array $k04Patterns): float {
    $presentIn  = 0;  // số hypothesis mà symptom này xuất hiện trong required/supporting
    $absentIn   = 0;  // số hypothesis mà symptom này trong differentiating_negative

    foreach ($hypotheses as $h) {
        $pattern = $k04Patterns[$h['pattern_id']] ?? null;
        if (!$pattern) continue;

        if (in_array($symptomCode, $pattern['required_symptoms'] ?? [])
         || in_array($symptomCode, $pattern['supporting_symptoms'] ?? [])) {
            $presentIn++;
        }
        if (in_array($symptomCode, $pattern['differentiating_symptoms_negative'] ?? [])) {
            $absentIn++;
        }
    }

    // Entropy cao nhất khi symptom chia đều hypotheses 50/50
    // Symptom chỉ present trong 1 hypothesis → thông tin tốt
    // Symptom present trong tất cả → thông tin kém
    $total = count($hypotheses);
    if ($total === 0 || ($presentIn + $absentIn) === 0) return 0;

    $ratio = $presentIn / $total;
    // Entropy = -p*log(p) - (1-p)*log(1-p), simplified
    if ($ratio === 0 || $ratio === 1) return 0;
    return -($ratio * log($ratio, 2)) - ((1-$ratio) * log(1-$ratio, 2));
}
```

## PHẦN 4: SEMANTIC SEARCH — Hệ thống Embedding Hybrid

### 4.1 Quyết định kiến trúc: Browser Embedding + Token Fallback

Expert 2 đề xuất vector embedding (pgvector — server-side). Tuy nhiên có giải pháp tốt hơn: embedding chạy hoàn toàn **trong browser** bằng Transformers.js, lưu binary vào SQLite, không cần server-side ML, không cần pgvector.

**So sánh phương án:**

| Phương án | Chi phí infra | Semantic coverage | Thời gian setup | Quyết định |
|-----------|--------------|-------------------|----------------|------------|
| Pure token-match | Không | ~60% | Đã có | Fallback khi embedding chưa sẵn |
| Token + Alias + Variants | Không | ~80% | 1 ngày | ✅ Luôn chạy song song |
| **Browser Embedding (Transformers.js)** | **Không** | **~92%** | **2-3 ngày** | **✅ Primary search** |
| pgvector (server ML) | Server GPU/API | ~95% | 2 tuần | Phase 3 nếu cần |

**Kiến trúc được chọn:**
```
Search pipeline (theo thứ tự ưu tiên):
  1. Embedding search (Transformers.js + Hamming distance)  ← Primary
     - Chạy trong browser, không cần server inference
     - Model: Xenova/all-MiniLM-L6-v2 (25MB, load 1 lần)
     - Search latency: <10ms cho 50k records
  2. Token-match + Alias search                             ← Fallback
     - Dùng khi model chưa load hoặc user không cho phép WebAssembly
     - Luôn chạy song song, merge kết quả
```

### 4.3 Binary Embedding Format

```
Model output: 384 float values (Xenova/all-MiniLM-L6-v2)
Binary conversion:
  - value > 0 → bit = 1
  - value ≤ 0 → bit = 0
  - Pack 384 bits → 48 bytes BLOB

Storage: 48 bytes / document
10,000 documents → 480 KB (dataset.bin)
50,000 documents → 2.4 MB (dataset.bin)
```

```php
<?php
/**
 * Build dataset.bin từ embedding_documents
 * Mỗi record: 48 bytes embedding + metadata reference
 * Output:
 *   - public/embeddings/dataset.bin   (48 bytes × N documents)
 *   - public/embeddings/metadata.json (array of {id, doc_type, source_id, text_preview})

define('APP_ROOT', __DIR__ . '/../app');
require_once APP_ROOT . '/core/Database.php';

$pdo = Database::get();

// Đọc theo doc_type để build separate indices
$docTypes = ['k02_symptom', 'k04_pattern', 'k04_question', 'disease_node'];

foreach ($docTypes as $type) {
    $rows = $pdo->query("
        SELECT id, source_id, text, embedding
        FROM embedding_documents
        WHERE doc_type = '$type' AND embedding IS NOT NULL
        ORDER BY id
    ")->fetchAll(PDO::FETCH_ASSOC);

    if (empty($rows)) continue;

    $binFile  = __DIR__ . "/../public/embeddings/{$type}.bin";
    $metaFile = __DIR__ . "/../public/embeddings/{$type}_meta.json";

    $fp   = fopen($binFile, 'wb');
    $meta = [];

    foreach ($rows as $row) {
        fwrite($fp, $row['embedding']);  // exactly 48 bytes
        $meta[] = [
            'id'       => $row['id'],
            'src'      => $row['source_id'],
            'preview'  => mb_substr($row['text'], 0, 80),
        ];
    }

    fclose($fp);
    file_put_contents($metaFile, json_encode($meta, JSON_UNESCAPED_UNICODE));
    echo "Built {$type}.bin: " . count($rows) . " vectors\n";
}
```

### 4.6 JavaScript — Embedding & Search Engine

**`public/js/semantic-search.js`**:

```javascript
import { pipeline } from 'https://cdn.jsdelivr.net/npm/@xenova/transformers@2/dist/transformers.min.js';

// ── Utilities ──────────────────────────────────────────

/** Convert float array (384 dim) → 48-byte Uint8Array (binary hash) */
function floatToBinary(floats) {
    const bits = new Uint8Array(48);
    for (let i = 0; i < 384; i++) {
        if (floats[i] > 0) bits[Math.floor(i / 8)] |= (1 << (7 - (i % 8)));
    }
    return bits;
}

/** Pack Uint8Array to base64 for API transmission */
function packBitsToBase64(bits) {
    return btoa(String.fromCharCode(...bits));
}

/** Precomputed popcount lookup table for XOR byte */
const POPCOUNT = new Uint8Array(256);
for (let i = 0; i < 256; i++) {
    let x = i; x = x - ((x >> 1) & 0x55); x = (x & 0x33) + ((x >> 2) & 0x33);
    POPCOUNT[i] = ((x + (x >> 4)) & 0x0F);
}

/** Hamming distance between two 48-byte vectors */
function hamming48(a, b, offsetA, offsetB) {
    let dist = 0;
    for (let i = 0; i < 48; i++) dist += POPCOUNT[a[offsetA + i] ^ b[offsetB + i]];
    return dist;
}

/** Top-K search in binary dataset */
function topKSearch(queryBits, dataset, k = 10) {
    const n = dataset.length / 48;
    const heap = [];  // max-heap by distance

    for (let i = 0; i < n; i++) {
        const dist = hamming48(queryBits, dataset, 0, i * 48);
        if (heap.length < k || dist < heap[0].dist) {
            heap.push({ idx: i, dist });
            heap.sort((a, b) => b.dist - a.dist);
            if (heap.length > k) heap.shift();
        }
    }
    return heap.sort((a, b) => a.dist - b.dist);
}

// ── SemanticSearch Class ──────────────────────────────

class SemanticSearch {
    constructor() {
        this.embedder  = null;
        this.datasets  = {};  // { doc_type: Uint8Array }
        this.metadata  = {};  // { doc_type: Array }
        this.modelReady = false;
    }

    async loadModel() {
        this.embedder = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');
        this.modelReady = true;
        console.log('Semantic model ready');
    }

    async loadIndex(docType) {
        const [binRes, metaRes] = await Promise.all([
            fetch(`/embeddings/${docType}.bin`),
            fetch(`/embeddings/${docType}_meta.json`),
        ]);
        this.datasets[docType] = new Uint8Array(await binRes.arrayBuffer());
        this.metadata[docType] = await metaRes.json();
    }

    async embed(text) {
        const output = await this.embedder(text, { pooling: 'mean', normalize: true });
        return floatToBinary(output.data);
    }

    /** Search trong một doc_type cụ thể */
    async search(query, docType, topK = 10) {
        if (!this.modelReady) throw new Error('Model not loaded');
        if (!this.datasets[docType]) await this.loadIndex(docType);

        const queryBits = await this.embed(query);
        const results   = topKSearch(queryBits, this.datasets[docType], topK);

        return results.map(r => ({
            ...this.metadata[docType][r.idx],
            similarity: 1 - r.dist / 384,  // normalize to 0–1
        }));
    }

    /** Multi-index search: merge results từ nhiều doc_type */
    async searchMulti(query, docTypes, topK = 5) {
        const results = await Promise.all(
            docTypes.map(t => this.search(query, t, topK))
        );
        return results.flat().sort((a, b) => b.similarity - a.similarity).slice(0, topK);
    }
}

// Singleton instance
window.semanticSearch = new SemanticSearch();
```

### 4.9 Token-match vẫn dùng làm Fallback

```php
// Mở rộng tokenMatchScore() — vẫn dùng khi embedding chưa load:
private function tokenMatchScore(array $tokens, array $sym, array $k01Data): float {
    $sourceA = $sym['name_vi'] ?? '';
    $sourceB = $sym['yhct_clinical_note'] ?? '';
    $sourceC = $sym['yhhd_clinical_note'] ?? '';
    $sourceD = implode(' ', $sym['aliases'] ?? []);
    $sourceE = $k01Data['variants_vi'][$sym['symptom_code']] ?? '';
    $sourceF = $k01Data['option_labels'][$sym['symptom_code']] ?? '';

    $score = 0;
    foreach ($tokens as $t) {
        $score += max(
            mb_strpos($sourceA, $t) !== false ? 1.00 : 0,
            mb_strpos($sourceD, $t) !== false ? 0.90 : 0,
            mb_strpos($sourceE, $t) !== false ? 0.85 : 0,
            mb_strpos($sourceB, $t) !== false ? 0.70 : 0,
            mb_strpos($sourceF, $t) !== false ? 0.60 : 0,
            mb_strpos($sourceC, $t) !== false ? 0.50 : 0   // yhhd note
        );
    }
    return count($tokens) > 0 ? $score / count($tokens) : 0;
}
```

## PHẦN 7: CO-OCCURRENCE MATRIX — Bổ sung K04 Weights

Expert 3 chỉ ra: co-occurrence chỉ từ K03 thưa, cần thêm K04 với weight cao hơn.

```php
function buildCoOccurrenceMatrix(array $k03Rules, array $k04Patterns): array {
    $coMatrix = [];

    // K03 pathogenesis level — weight 1.0
    foreach ($k03Rules as $rule) {
        $codes = array_merge(
            $rule['required_symptoms'] ?? [],
            $rule['supporting_symptoms'] ?? []
        );
        foreach ($codes as $a) {
            foreach ($codes as $b) {
                if ($a !== $b) $coMatrix[$a][$b] = ($coMatrix[$a][$b] ?? 0) + 1.0;
            }
        }
    }

    // K04 pattern level — weight cao hơn vì gần output nhất
    foreach ($k04Patterns as $pattern) {
        $required = $pattern['required_symptoms'] ?? [];
        $twoOf    = $pattern['two_or_more_of'] ?? [];

        // Required symptoms: signal cực mạnh
        foreach ($required as $a) {
            foreach ($required as $b) {
                if ($a !== $b) $coMatrix[$a][$b] = ($coMatrix[$a][$b] ?? 0) + 2.0;
            }
        }
        // two_or_more_of: signal trung bình
        foreach ($twoOf as $a) {
            foreach ($twoOf as $b) {
                if ($a !== $b) $coMatrix[$a][$b] = ($coMatrix[$a][$b] ?? 0) + 1.5;
            }
        }
    }

    return $coMatrix;
}
```
