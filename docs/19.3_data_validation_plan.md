## PHẦN 1: KNOWLEDGE BASE SIZE REQUIREMENTS

### 1.1 Vấn đề: KB hiện tại đủ không?

Expert 1: "Hiện tại bạn mới đạt khoảng 15-20% quy mô cần thiết."
Expert 3: "KB đã có 180 K03 rules — co-occurrence matrix là workable. Vấn đề không phải thiếu data, mà PRD-16 chưa khai thác hết sức mạnh."

**Phân tích thực tế:**

|----|---------|---------------------|---------------|-----------|

| K02 Symptoms | 300–345 | 300 | 600+ | ✅ Đủ cho MVP |
| K03 Pathogenesis rules | 180 | 80 | 150+ | ✅ Vượt tối thiểu |
| K04 Patterns | 135 | 120 | 250+ | ✅ Đủ cho MVP |
| K05 Red flags | 70 | 47 | 100+ | ✅ Đủ cho MVP |
| K06 Clusters | 60 | 30 | 60+ | ✅ Đạt |
| K07 Herb-drug | 80–120 | 80 | 200+ | ✅ Đủ cho MVP |
| K08 Kiêm chứng | 80 | 30 | 80+ | ✅ Đạt |
| **Disease Graph** | 0 | 50 | 200+ | ❌ Chưa có |
| **Disease Stages** | 0 | 5 bệnh | 50 bệnh | ❌ Chưa có |

**Kết luận:** KB hiện tại đủ để khởi động engine. Điểm thiếu duy nhất là Disease Graph (bệnh nền) — mới hoàn toàn, cần seed.

### 1.2 Tại sao co-occurrence matrix "workable" với KB hiện tại

```
180 K03 rules × trung bình 6 symptoms/rule
= ~1080 symptom-in-rule instances

Sau khi build co-matrix với K04 weights (19.1 §7):
180 K03 rules × 2 = + 135 K04 patterns × 1.5–2.0
= ~1,600+ co-occurrence signal points

Với 345 K02 symptoms: matrix 345×345 = 119,025 cells
~1,600 non-zero cells = ~1.3% density

Tối thiểu cần: ~0.5% density để có ý nghĩa → ✅ Đạt
Lý tưởng: ~3% density → Cần bổ sung KB về sau
```

### 1.3 Ngưỡng KB để features hoạt động có ý nghĩa

| Feature | Ngưỡng tối thiểu | Ngưỡng tốt |
|---------|-----------------|-----------|
| Co-occurrence re-ranking | 80 K03 rules | 150+ K03 rules |
| Differential (Nhóm 3) | 50 K04 patterns với differentiates_from | 100+ patterns |
| BackwardReasoningEngine | 50 disease nodes + 100 edges | 200+ nodes/edges |
| K07 drug check | 40 interactions (8 drug classes) | 80+ interactions |
| K06 time-gated | 3 diseases với stage data | 10+ diseases |

## PHẦN 1.5: EMBEDDING PIPELINE — Quy Trình Sinh Embedding

### 1.5.1 Tổng quan

Sau khi import KB records vào SQLite, cần sinh embedding cho các records để enable semantic search. Quy trình này chạy **một lần** sau import, và chạy lại khi data thay đổi.

```
Import KB records (K01/K02/K04/disease_nodes)
         ↓
scripts/seed_embedding_documents.php
  → Tạo rows trong embedding_documents
  → text = ghép các fields theo doc_type (xem §4.2 của 19.1)
  → embedding = NULL ban đầu
         ↓
Admin: /admin/embeddings → "Generate Missing"
  → Browser load Xenova/all-MiniLM-L6-v2 (~25MB, WebAssembly)
  → Lần lượt embed từng document (~55ms/doc)
  → POST /api/embeddings/store với binary vector 48 bytes
  → Progress bar hiển thị tiến độ
         ↓
"Build Index" button
  → scripts/build_index.php
  → Output: public/embeddings/{type}.bin + {type}_meta.json
         ↓
Semantic search sẵn sàng
```

### 1.5.2 Ước tính thời gian sinh embedding

| doc_type | Số documents | Thời gian (~55ms/doc) |
|----------|-------------|----------------------|
| k02_symptom | 345 | ~19 phút |
| k04_pattern | 135 | ~7 phút |
| k01_phrase | 300 | ~16 phút |
| k04_question | ~270 (2 câu/pattern) | ~15 phút |
| disease_node | 50 | ~3 phút |
| **Tổng Sprint 1** | **~780** | **~43 phút** |
| Sau Sprint 2 thêm | ~200 | ~11 phút |

**Ghi chú:** Quá trình này chạy một lần trong browser của admin. Browser tab phải giữ mở. Nên chạy vào buổi tối.

### 1.5.3 Script: `scripts/seed_embedding_documents.php`

```php
<?php
/**
 * Populate embedding_documents từ K01/K02/K04/disease_nodes
 * Chạy sau mỗi lần import KB batch mới
 */
define('APP_ROOT', __DIR__ . '/../app');
require_once APP_ROOT . '/core/Database.php';

$pdo = Database::get();

// K02 symptoms
$stmt = $pdo->query("SELECT symptom_code, name_vi, yhct_clinical_note, yhhd_clinical_note FROM kb_symptoms WHERE status='active'");
$insertStmt = $pdo->prepare("
    INSERT OR IGNORE INTO embedding_documents (doc_type, source_id, source_table, text)
    VALUES (?, ?, ?, ?)
");
foreach ($stmt->fetchAll(PDO::FETCH_ASSOC) as $row) {
    $text = trim(implode(' ', array_filter([
        $row['name_vi'],
        $row['yhct_clinical_note'],
        $row['yhhd_clinical_note'],
    ])));
    $insertStmt->execute(['k02_symptom', $row['symptom_code'], 'kb_symptoms', $text]);
}

// K04 patterns
$stmt = $pdo->query("SELECT chung_code, name_vi, clinical_note FROM kb_patterns WHERE status='active'");
foreach ($stmt->fetchAll(PDO::FETCH_ASSOC) as $row) {
    $text = trim("{$row['name_vi']} {$row['clinical_note']}");
    $insertStmt->execute(['k04_pattern', $row['chung_code'], 'kb_patterns', $text]);
}

// K04 key questions (từ JSON field)
$stmt = $pdo->query("SELECT chung_code, key_questions FROM kb_patterns WHERE key_questions IS NOT NULL");
foreach ($stmt->fetchAll(PDO::FETCH_ASSOC) as $row) {
    $questions = json_decode($row['key_questions'], true) ?? [];
    foreach ($questions as $i => $q) {
        $text = $q['question_vi'] . ' ' . ($q['symptom_code'] ?? '');
        $insertStmt->execute(['k04_question', $row['chung_code'] . '_q' . $i, 'kb_patterns', $text]);
    }
}

// K01 observable phrases
$stmt = $pdo->query("SELECT phrase_id, phrase_vi, variants_vi FROM kb_observable_phrases");
foreach ($stmt->fetchAll(PDO::FETCH_ASSOC) as $row) {
    $variants = implode(' ', json_decode($row['variants_vi'] ?? '[]', true));
    $text = trim($row['phrase_vi'] . ' ' . $variants);
    $insertStmt->execute(['k01_phrase', $row['phrase_id'], 'kb_observable_phrases', $text]);
}

echo "Done. Total embedding_documents: " .
     $pdo->query("SELECT COUNT(*) FROM embedding_documents")->fetchColumn() . "\n";
echo "Pending embeddings: " .
     $pdo->query("SELECT COUNT(*) FROM embedding_documents WHERE embedding IS NULL")->fetchColumn() . "\n";
```

### 1.5.4 Thêm vào migration schema

```sql
-- Thêm vào scripts/migrate_embeddings.php

CREATE TABLE IF NOT EXISTS embedding_documents (
    id                   INTEGER PRIMARY KEY AUTOINCREMENT,
    doc_type             TEXT NOT NULL,
    source_id            TEXT NOT NULL,
    source_table         TEXT NOT NULL,
    text                 TEXT NOT NULL,
    embedding            BLOB,
    embedding_updated_at DATETIME,
    UNIQUE(doc_type, source_id)
);
CREATE INDEX IF NOT EXISTS idx_emb_type ON embedding_documents(doc_type);
CREATE INDEX IF NOT EXISTS idx_emb_pending ON embedding_documents(doc_type) WHERE embedding IS NULL;
```

## PHẦN 2: LLM DATA VALIDATION — PHƯƠNG PHÁP BÀI BẢN

### 2.1 Vấn đề: LLM Hallucination trong Dữ Liệu Y Khoa

Expert 2: "Dữ liệu y khoa do AI sinh ra hàng loạt chắc chắn chứa Hallucination chết người. Nếu bạn import thẳng vào DB và chỉ duyệt qua loa trên Dashboard, hệ thống sẽ giết người trước khi kịp chữa bệnh."

**Ví dụ hallucination nguy hiểm:**
- Map "mồ hôi trộm" sang Pháp trị "Phát hãn" thay vì "Tư âm" (ngược hoàn toàn)
- K05: Đặt sai triage level (L3 thay vì L1 cho meningitis)
- K07: Miss một tương tác nguy hiểm (Đan sâm + Warfarin)
- K04: required_symptoms bao gồm triệu chứng không liên quan

### 2.2 Chiến lược Review — 3 Tầng

```
Tầng 1: Automated Structural Validation (chạy tự động, không cần người)
  → Schema validation (field bắt buộc, format, unique keys)
  → Cross-reference validation (symptom_codes tồn tại trong K02)
  → Statistical anomaly detection (weight outliers, empty arrays)
  Pass rate kỳ vọng: 85-90% records

Tầng 2: Automated Clinical Simulation (chạy tự động, output cho bác sĩ review)
  → 200+ test cases đi qua engine với KB hiện tại
  → So sánh output với "expected" (xem §2.3)
  → Flag records gây sai output
  Thời gian bác sĩ cần: review 200 test outputs, không phải 1185 JSON records

Tầng 3: Manual Review (bác sĩ chuyên gia)
  → Chỉ review records bị flag từ Tầng 1+2
  → Ưu tiên theo safety impact (K05 > K07 > K04 > K03 > K01 > K02)
  → Estimated: 40-60 hours tổng (không phải review 1185 records từng cái)
```

### 2.4 Script: `scripts/run_clinical_tests.php`

```php
<?php
/**
 * Clinical Test Runner — chạy 200 test cases qua engine
 * Báo cáo: pass/fail, metrics, flagged records
 */

define('APP_ROOT', __DIR__ . '/../app');
require_once APP_ROOT . '/core/Database.php';
// ... requires

$testSuite = json_decode(file_get_contents(__DIR__ . '/test_cases.json'), true);
$results   = [];
$flagged   = [];

foreach ($testSuite as $tc) {
    // Chạy engine với input của test case
    $engineResult = runEngine($tc['input_symptoms'], $tc['context_flags'] ?? []);

    // Kiểm tra expected
    $pass = evaluateTestCase($tc, $engineResult);

    $results[] = [
        'test_id'    => $tc['test_id'],
        'pass'       => $pass,
        'expected'   => $tc['expected'],
        'got'        => [
            'primary_pattern' => $engineResult['chung_ranked'][0]['pattern']['chung_code'] ?? null,
            'triage_level'    => $engineResult['triage_level'] ?? null,
            'score'           => $engineResult['chung_ranked'][0]['score'] ?? 0,
        ],
    ];

    if (!$pass) {
        // Flag records có thể gây lỗi để review
        $flagged[] = [
            'test_id'  => $tc['test_id'],
            'issue'    => detectIssueSource($tc, $engineResult),
            'review_k' => getRelevantKBRecords($tc, $engineResult),
        ];
    }
}

// Generate report
generateReport($results, $flagged, $outputFormat);
```

### 7.2 JavaScript — Admin Embedding Generator

```javascript
// public/js/admin-embeddings.js

import { pipeline } from 'https://cdn.jsdelivr.net/npm/@xenova/transformers@2/dist/transformers.min.js';

let embedder = null;
let cancelled = false;

async function loadModel(onProgress) {
    embedder = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2', {
        progress_callback: onProgress,
    });
}

async function generateEmbeddings(docType) {
    let offset = 0;
    const limit = 50;
    cancelled = false;

    while (!cancelled) {
        const resp = await fetch(`/admin/api/embedding-pending?doc_type=${docType}&limit=${limit}&offset=${offset}`);
        const docs = await resp.json();
        if (docs.length === 0) break;

        for (const doc of docs) {
            if (cancelled) return;

            // Embed text
            const output = await embedder(doc.text, { pooling: 'mean', normalize: true });
            const floats = Array.from(output.data);

            // Convert to 48-byte binary
            const bits = new Uint8Array(48);
            for (let i = 0; i < 384; i++) {
                if (floats[i] > 0) bits[Math.floor(i/8)] |= (1 << (7 - (i%8)));
            }

            // Store via API
            await fetch('/admin/api/embedding-store', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({
                    id: doc.id,
                    embedding_b64: btoa(String.fromCharCode(...bits)),
                }),
            });

            updateProgress(doc.id, doc.text.substring(0, 60));
        }
        offset += limit;
    }
}

async function buildIndex() {
    const resp = await fetch('/admin/api/build-index', { method: 'POST' });
    const result = await resp.json();
    showSuccess(`Built ${result.files_built.join(', ')} — ${result.total_vectors} vectors`);
}
```
