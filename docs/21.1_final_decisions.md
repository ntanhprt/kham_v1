### 1.2 Giải pháp đúng: Lưu vector riêng biệt theo provider

### 1.3 Logic quan trọng: Chỉ sinh lại khi cần

```
Khi admin nhấn "Generate Missing":
  Với mỗi document chưa có vector cho active_provider:
    → Sinh vector mới
    → Lưu vào embedding_vectors với content_hash

Khi admin nhấn "Regenerate Changed":
  Với mỗi document đã có vector cho active_provider:
    SHA256(document.text) ≠ embedding_vectors.content_hash?
    → Yes: Sinh lại (content thay đổi)
    → No: Bỏ qua (không cần sinh lại)

Khi chuyển đổi provider (VD: browser → openai):
  1. Đổi system_settings.embedding_active_provider
  2. Chỉ cần generate documents chưa có vector với provider mới
  3. Không xóa vector của provider cũ (tiết kiệm, có thể switch lại)
  4. Search index (.bin) được rebuild từ embedding_vectors WHERE provider = active_provider
```

**Lợi ích:**
- Không bao giờ mất dữ liệu embedding cũ khi đổi provider
- Không cần regenerate 780 docs nếu chỉ thêm vài docs mới
- Có thể A/B test 2 providers trên cùng search query
- Phát hiện tự động khi KB text bị sửa → chỉ regenerate record đó

### 1.4 Float32 thay vì Binary (sửa từ E3.2)

```javascript
// TRƯỚC (sai cho dataset nhỏ):
// Binary 48 bytes, Hamming distance — mất 15% recall

// SAU (đúng):
const DIMS = 384;  // hoặc 768/1024/1536 tùy provider

// dataset.bin format: Float32Array (mỗi vector = DIMS × 4 bytes)
// 780 docs × 384 dims × 4 bytes = 1.2 MB — load được vào browser hoàn toàn

function cosineSimilarity(a, b, offsetA, offsetB) {
    let dot = 0, normA = 0, normB = 0;
    for (let i = 0; i < DIMS; i++) {
        const ai = a[offsetA + i], bi = b[offsetB + i];
        dot   += ai * bi;
        normA += ai * ai;
        normB += bi * bi;
    }
    if (normA === 0 || normB === 0) return 0;
    return dot / (Math.sqrt(normA) * Math.sqrt(normB));
}

// Linear scan 780 docs × 384 dims: ~3ms — không cần binary trick
function topKSearch(queryVec, dataset, metadata, k = 10) {
    const n = dataset.length / DIMS;
    const scores = [];
    for (let i = 0; i < n; i++) {
        scores.push({
            idx: i,
            similarity: cosineSimilarity(queryVec, dataset, 0, i * DIMS),
            ...metadata[i]
        });
    }
    return scores.sort((a, b) => b.similarity - a.similarity).slice(0, k);
}
```

## PHẦN 2: VẤN ĐỀ CỐT LÕI MỚI — KHOẢNG TRỐNG NGỮ NGHĨA Y KHOA

### 2.1 Vấn đề: Semantic Similarity ≠ Medical Relevance

Đây là vấn đề **quan trọng nhất chưa được giải quyết** trong 19.1/19.2/19.3.

Semantic search tìm thứ "nghĩa gần nhau". Nhưng trong y học, nhiều mối liên hệ **không có nghĩa gần nhau về ngôn ngữ**:

```
Loại 1 — Nhân quả / Trigger:
  "uống bia rượu tối qua"    → đau đầu, buồn nôn sáng hôm sau
  "bỏ ăn sáng"               → run tay, chóng mặt, hoa mắt (hạ đường huyết)
  "ăn hải sản"               → nổi mẩn ngứa, sưng môi (dị ứng)
  "ngồi lâu 8 tiếng"         → tê bì chân, đau mỏi thắt lưng
  "căng thẳng deadline"       → đau dạ dày, đau đầu, mất ngủ

Loại 2 — Thời gian / Chu kỳ:
  "ngày đèn đỏ / hành kinh"  → đau bụng dưới, đau lưng, đau ngực, bứt rứt
  "trước kỳ kinh 3-5 ngày"   → đau đầu, phù chân tay, dễ cáu (PMS)
  "mới thức dậy buổi sáng"   → cứng khớp tay (viêm khớp dạng thấp)
  "sau khi ăn 30 phút"        → đầy bụng, ợ hơi, khó chịu vùng thượng vị

Loại 3 — Nhân khẩu / Ngữ cảnh:
  "đang mang thai tháng thứ 3"→ buồn nôn, mệt mỏi, chóng mặt, đau lưng
  "vừa sinh con xong"         → rụng tóc, mệt mỏi, dễ khóc (trầm cảm sau sinh)
  "người già nằm liệt giường" → loét da, phù chân, viêm phổi
  "trẻ em dưới 6 tuổi sốt"   → co giật do sốt cao

Loại 4 — Đau phản chiếu / Atypical:
  Nhồi máu cơ tim ACS        → đau vai trái, đau hàm, buồn nôn (không đau ngực)
  Túi mật viêm               → đau vai phải (không phải đau bụng phải)
  Viêm ruột thừa giai đoạn đầu → đau quanh rốn (không phải hố chậu phải)
  Loét dạ dày                → đau lưng giữa (không phải đau bụng)

Loại 5 — Hệ thống / Bệnh nền:
  Tiểu đường                 → vết thương lâu lành, hay nhiễm trùng, khát nước
  Suy giáp                   → mệt mỏi, lạnh, táo bón, tóc rụng, tăng cân
  Thiếu máu thiếu sắt        → mệt mỏi, xanh xao, khó thở khi gắng sức, móng giòn

Loại 6 — YHCT đặc thù tiếng Việt:
  "nóng trong người"          → nhiệt miệng, mụn, táo bón, tiểu vàng
  "bị phong"                  → méo miệng, tê liệt nửa người (stroke)
  "bị hư"                     → mệt mỏi, lưng mỏi gối yếu (Thận hư)
  "bị lạnh bụng"              → đau bụng, tiêu lỏng (Tỳ hư hàn)
```

**Semantic search KHÔNG tự tìm được những liên hệ này** vì `uống bia` và `đau đầu` không có nghĩa ngữ nghĩa gần nhau trong bất kỳ embedding model nào.

### 2.2 Kiến Trúc Giải Quyết: 3 Tầng Bổ Sung

```
Tầng 1 (ĐÃ CÓ):     Semantic Search — tìm theo nghĩa gần
                      "bị nặng đầu" → headache symptoms

Tầng 2 (THÊM MỚI):  Context Trigger Layer — tìm theo trigger/ngữ cảnh
                      "uống bia hôm qua" → hangover → headache, nausea

Tầng 3 (THÊM MỚI):  Enriched Embedding Text — embedding biết ngữ cảnh
                      Text của "đau đầu" bao gồm: "sau uống rượu", "do kỳ kinh"
                      → Semantic search cũng có thể tìm được
```

### 3.2 Phase 0 Context Extractor — Mở Rộng

Trong `app/engine/Phase0Parser.php`, bổ sung hàm `extractContextTriggers()`:

```php
/**
 * Trích xuất triggers từ chief complaint
 * Chạy song song với tokenize và K01 match
 */
public function extractContextTriggers(string $rawInput): array {
    $triggers = [];

    // Load kb_symptom_contexts.context_phrase từ cache
    $phrases = $this->loadContextPhrases();

    $lower = mb_strtolower($rawInput);
    foreach ($phrases as $phrase => $data) {
        // Exact substring match trên input thô
        if (mb_strpos($lower, mb_strtolower($phrase)) !== false) {
            foreach ($data as $entry) {
                $triggers[] = [
                    'symptom_code'   => $entry['symptom_code'],
                    'context_phrase' => $phrase,
                    'context_type'   => $entry['context_type'],
                    'boost_weight'   => $entry['boost_weight'],
                ];
            }
        }
    }

    // Normalize: mỗi symptom_code lấy boost cao nhất nếu có nhiều match
    $result = [];
    foreach ($triggers as $t) {
        $code = $t['symptom_code'];
        if (!isset($result[$code]) || $result[$code]['boost_weight'] < $t['boost_weight']) {
            $result[$code] = $t;
        }
    }

    return array_values($result);
}
```

## PHẦN 5: FIX ENTROPY VÀ HYPOTHESIS RANKING

### 5.1 Entropy với Prevalence Weighting (sửa từ E1.2 + E2.4)

```php
/**
 * Entropy có trọng số Prevalence — Bayes-aware
 * Câu hỏi tốt = câu hỏi phân chia "khối lượng xác suất" tốt nhất,
 * không phải câu chia đều số lượng bệnh
 */
function calculateWeightedEntropyScore(
    string $symptomCode,
    array $hypotheses,
    array $k04Patterns,
    array $diseasePrevalence  // prevalence_vn từ disease_nodes
): float {
    // Tính prior probability cho mỗi hypothesis
    $priorMap = [
        'high'   => 0.40,
        'medium' => 0.20,
        'low'    => 0.08,
        'rare'   => 0.02,
    ];

    $totalPrior = 0;
    $priors = [];
    foreach ($hypotheses as $h) {
        $prev = $diseasePrevalence[$h['pattern_id']] ?? 'low';
        $prior = $priorMap[$prev] ?? 0.08;
        $priors[$h['pattern_id']] = $prior;
        $totalPrior += $prior;
    }

    // Normalize priors
    foreach ($priors as $pid => $p) {
        $priors[$pid] = $totalPrior > 0 ? $p / $totalPrior : 1 / count($hypotheses);
    }

    // Tính P(symptom present | hypothesis) = weighted probability mass
    $pPresent = 0;
    foreach ($hypotheses as $h) {
        $pattern = $k04Patterns[$h['pattern_id']] ?? null;
        if (!$pattern) continue;

        $inRequired   = in_array($symptomCode, $pattern['required_symptoms'] ?? []);
        $inSupporting = in_array($symptomCode, $pattern['supporting_symptoms'] ?? []);

        if ($inRequired || $inSupporting) {
            $pPresent += $priors[$h['pattern_id']];
        }
    }

    // Entropy = -p*log2(p) - (1-p)*log2(1-p)
    if ($pPresent <= 0 || $pPresent >= 1) return 0.0;
    return -($pPresent * log($pPresent, 2))
           - ((1 - $pPresent) * log(1 - $pPresent, 2));
}
```

**Ví dụ ảnh hưởng:**
```
Trước đây: Cảm cúm (10,000 người) vs Viêm màng não (1 người)
→ Engine chia đều 50/50 → hỏi câu loại trừ viêm màng não liên tục

Sau sửa:
Prior: Cảm cúm 0.95, Viêm màng não 0.05
→ Engine ưu tiên confirm Cảm cúm trước
→ Chỉ hỏi câu viêm màng não khi có red flag triggers hoặc user già/suy giảm miễn dịch
```

```javascript
        for (const doc of docs) {
            if (cancelled) {
                // Lưu checkpoint trước khi dừng
                saveCheckpoint(docType, offset, processedIds);
                showInfo('Paused. Progress saved — resume anytime.');
                return;
            }

            if (processedIds.has(doc.id)) continue;  // Đã làm rồi

            const output = await embedder(doc.text, { pooling: 'mean', normalize: true });
            const floats = new Float32Array(output.data);

            // Convert to base64 for storage
            const buffer = floats.buffer;
            const b64 = btoa(String.fromCharCode(...new Uint8Array(buffer)));

            await fetch('/admin/api/embedding-store', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({ id: doc.id, vector_b64: b64, provider: activeProvider }),
            });

            processedIds.add(doc.id);
            updateProgress(doc.id, doc.text.substring(0, 60));

            // Save checkpoint mỗi 10 docs
            if (processedIds.size % 10 === 0) {
                saveCheckpoint(docType, offset, processedIds);
            }
        }
        offset += limit;
    }
}

function saveCheckpoint(docType, offset, processedIds) {
    localStorage.setItem(CHECKPOINT_KEY, JSON.stringify({
        offset,
        processed_ids: [...processedIds],
        saved_at: new Date().toISOString()
    }));
}
```

## PHẦN 9: BẢO MẬT — API KEY ENCRYPTION

```php
// app/core/SystemSettings.php

class SystemSettings {
    private static function getEncKey(): string {
        $key = $_ENV['SETTINGS_ENC_KEY'] ?? '';
        if (empty($key)) {
            throw new \RuntimeException('SETTINGS_ENC_KEY not set in .env');
        }
        return $key;
    }

    public static function set(string $key, string $value): void {
        if (str_ends_with($key, '_api_key') || str_ends_with($key, '_encrypted')) {
            $iv = openssl_random_pseudo_bytes(16);
            $encrypted = openssl_encrypt(
                $value, 'AES-256-CBC',
                hash('sha256', self::getEncKey(), true),
                OPENSSL_RAW_DATA, $iv
            );
            $value = base64_encode($iv . $encrypted);
        }
        // Save to DB...
    }

    public static function get(string $key): string {
        $raw = self::getRaw($key);
        if (str_ends_with($key, '_api_key') || str_ends_with($key, '_encrypted')) {
            $decoded = base64_decode($raw);
            $iv      = substr($decoded, 0, 16);
            $data    = substr($decoded, 16);
            return openssl_decrypt(
                $data, 'AES-256-CBC',
                hash('sha256', self::getEncKey(), true),
                OPENSSL_RAW_DATA, $iv
            ) ?: '';
        }
        return $raw;
    }
}
```
