# 1. Cấu trúc ontology tổng thể của YHCT

Toàn bộ reasoning của YHCT có thể mô hình hóa thành **7 tầng logic**:

```
OBSERVABLES
↓
SYMPTOMS
↓
BÁT CƯƠNG
↓
TẠNG PHỦ
↓
BỆNH CƠ (Pathogenesis)
↓
CHỨNG (Pattern)
↓
PHÁP TRỊ
```

Trong database / knowledge graph:

```
Observation → Symptom → Pattern features → Pattern → Treatment principle
```

---

# 2. Layer 1 — Observables (dữ liệu bệnh nhân nói)

Đây là **dữ liệu thô bệnh nhân mô tả**.

Ví dụ:

```
đau đầu
chóng mặt
nóng trong
đầy bụng
khó ngủ
mệt mỏi
```

Database:

```
observable_id
patient_vocabulary
clinical_code
category
```

Ví dụ bảng:

| observable | code                 |
| ---------- | -------------------- |
| nóng trong | heat_internal        |
| nặng đầu   | head_heaviness       |
| tức ngực   | chest_pressure       |
| đầy bụng   | abdominal_distension |

---

# 3. Layer 2 — Symptoms (triệu chứng chuẩn hóa)

Một observable có thể map thành nhiều **symptom lâm sàng**.

Ví dụ:

```
"nóng trong"
↓
internal_heat
dry_mouth
thirst
```

Schema:

```
observable_symptom_mapping

observable
symptom
confidence
```

Ví dụ:

| observable | symptom       |
| ---------- | ------------- |
| nóng trong | internal_heat |
| nóng trong | thirst        |
| nóng trong | mouth_dryness |

---

# 4. Layer 3 — Bát Cương Inference

Đây là **bước reasoning đầu tiên của YHCT**.

8 dimension:

```
Hàn – Nhiệt
Hư – Thực
Biểu – Lý
Âm – Dương
```

Nhưng trong machine reasoning nên encode thành **6 dimension thực dụng**:

```
cold
heat
deficiency
excess
exterior
interior
```

Schema:

```
symptom_bac_cuong_mapping
```

Ví dụ:

| symptom      | dimension  | weight |
| ------------ | ---------- | ------ |
| thirst       | heat       | 0.6    |
| fear_cold    | cold       | 0.8    |
| fatigue      | deficiency | 0.7    |
| irritability | excess     | 0.5    |

Inference:

```
score_heat = Σ symptom weights
score_cold = Σ symptom weights
```

---

# 5. Layer 4 — Tạng Phủ Mapping

Triệu chứng được map sang **tạng phủ**.

Các tạng chính:

```
Can
Tâm
Tỳ
```

Schema:

```
symptom_zangfu_mapping
```

Ví dụ:

| symptom      | organ  | weight |
| ------------ | ------ | ------ |
| headache     | liver  | 0.6    |
| irritability | liver  | 0.7    |
| insomnia     | heart  | 0.6    |
| fatigue      | spleen | 0.7    |
| tinnitus     | kidney | 0.8    |

# 6. Layer 5 — Bệnh Cơ (Pathogenesis)

Đây là **tầng quan trọng nhất của reasoning YHCT**.

Tạng phủ + Bát cương → **cơ chế bệnh**.

Ví dụ:

```
liver + heat → liver fire
liver + deficiency → liver yin deficiency
spleen + deficiency → spleen qi deficiency
```

Danh mục bệnh cơ nên có khoảng **60–80 loại**.

Ví dụ:

```

spleen_yang_deficiency
kidney_yin_deficiency

blood_stasis
qi_deficiency
```

Schema:

```
pathogenesis_rules

organ
bac_cuong_pattern
→ pathogenesis
```

# 10. Reasoning pipeline của AI

Engine chạy theo pipeline:

```
1 collect symptoms
2 infer bac_cuong
3 infer zangfu
4 infer pathogenesis
5 infer pattern
6 suggest treatment principle
```

Pseudo logic:

```
symptoms = collect()

bac_cuong = infer_bac_cuong(symptoms)

organs = infer_zangfu(symptoms)

pathogenesis = infer_pathogenesis(organs, bac_cuong)

pattern = infer_pattern(pathogenesis)

treatment = lookup(pattern)
```
