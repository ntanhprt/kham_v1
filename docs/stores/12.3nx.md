# 1. Thiếu tầng “Temporal reasoning” (logic thời gian)

Bác sĩ giỏi **không chỉ hỏi triệu chứng**, họ hỏi **timeline**.

Ví dụ:

```
đau đầu
```

chưa đủ.

Phải biết:

```
khởi phát khi nào
tăng dần hay giảm
có chu kỳ không
liên quan sự kiện gì
```

Ví dụ thực tế:

```
đau đầu + tăng chiều
```

có thể là:

```
tension headache
hypertension
liver yang rising
```

nhưng nếu:

```
đau đầu + sáng sớm + buồn nôn
```

→ bác sĩ sẽ nghĩ ngay:

```
tăng áp lực nội sọ
```

PRD của bạn có:

```
onset
duration
severity
```

nhưng **chưa có temporal pattern**.

Bạn cần thêm ontology:

```
temporal_pattern
```

Ví dụ:

| pattern             | nghĩa    |
| ------------------- | -------- |
| sudden onset        | cấp tính |
| gradual progression | tăng dần |
| episodic            | từng cơn |
| cyclical            | chu kỳ   |
| morning dominant    | sáng     |
| afternoon dominant  | chiều    |
| nocturnal           | ban đêm  |

Temporal reasoning cực kỳ quan trọng trong chẩn đoán.

# 2. Thiếu “severity trajectory”

Bác sĩ không chỉ hỏi:

```
severity = 6/10
```

Họ hỏi:

```
hôm qua bao nhiêu
hôm nay bao nhiêu
```

Ví dụ:

```
đau bụng 3 → 8
```

→ **appendicitis risk**

nhưng:

```
đau bụng 8 → 3
```

→ có thể **spasm / gas**

Hệ thống cần một biến:

```
symptom_trajectory
```

ví dụ:

```
improving
stable
worsening
fluctuating
```

Điều này ảnh hưởng lớn đến triage.

---

# 3. Thiếu “exposure reasoning”

Một bác sĩ giỏi luôn hỏi:

```
gần đây có gì thay đổi không?
```

Ví dụ:

```
ăn đồ lạ
du lịch
thuốc mới
stress
chấn thương
```

Những thứ này gọi là:

```
exposures
```

Bạn nên có module nhỏ:

```
Recent Changes (7–14 days)
```

Ví dụ:

```
Bạn có:
○ đổi thuốc gần đây
○ ăn thực phẩm lạ
○ đi du lịch
○ stress lớn
○ nhiễm bệnh gần đây
```

Điều này giúp reasoning cực nhiều.

---

# 4. Thiếu “negative findings”

Bác sĩ giỏi **luôn hỏi để loại trừ bệnh nguy hiểm**.

Ví dụ đau đầu:

bác sĩ sẽ hỏi:

```
có nôn không
có yếu tay chân không
có sốt không
```

Nếu bệnh nhân trả lời **KHÔNG**, đó là thông tin cực kỳ quan trọng.

Trong y khoa gọi là:

```
pertinent negatives
```

PRD hiện tại thu thập symptom **dương tính** khá tốt.

Nhưng cần một bảng:

```
critical_negatives
```

Ví dụ:

```
headache
→ no fever
→ no neurological deficit
→ no neck stiffness
```

Điều này tăng **độ tin cậy lâm sàng rất nhiều**.

---

# 5. Thiếu “symptom hierarchy”

Trong thực tế khám bệnh:

không phải mọi triệu chứng **đều quan trọng như nhau**.

Ví dụ:

```
đau đầu
khó ngủ
khát nước
```

Bác sĩ giỏi sẽ hỏi:

```
triệu chứng nào xuất hiện trước
triệu chứng nào nặng nhất
```

Bạn đã có **anchor symptom**, nhưng nên thêm:

```
symptom_priority
```

Ví dụ:

```
primary
secondary
associated
```

Điều này giúp reasoning rõ ràng hơn.

# 6. Cluster engine của bạn rất tốt nhưng còn thiếu “probabilistic weight”

Cluster hiện tại:

```
min_count = 3
```

Nhưng thực tế:

```
meningitis triad
```

không phải lúc nào cũng đủ 3.

Có khi chỉ:

```
headache + neck stiffness
```

đã đáng nghi.

Bạn nên cho cluster:

```
probability weight
```

ví dụ:

```
headache + neck stiffness = 0.7
+ photophobia = 0.9
```

Như vậy cluster engine **mềm hơn**.

# 7. Constitution module rất tốt nhưng cần một chỉnh sửa nhỏ

Constitution **không nên hỏi trước session**.

Trong thực tế YHCT:

```
thể chất
```

thường được nhận ra **sau khi hỏi triệu chứng**.

Vì vậy tốt hơn:

```
Phase 5B
```

sử dụng câu hỏi constitution.

Điều này giảm:

```
drop-off rate
```

# 8. Một thứ cực kỳ quan trọng bạn đã làm đúng

Integration Safety Filter của bạn **rất chuẩn**. 

Đây là thứ rất nhiều health AI **không nghĩ tới**.

Ví dụ bạn đưa:

```
YHHD urgent → suppress YHCT treatment
```

Đây là **clinical governance đúng**.
