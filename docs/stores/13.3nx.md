Mình đã đọc **toàn bộ PRD v3.0 rất kỹ**. Trước khi đưa gợi ý mới, mình phải nói rõ:

**PRD v3 đã đạt mức rất hiếm thấy đối với một hệ thống khám bệnh AI.**

Nó đã có:

* reasoning lâm sàng
* ontology YHCT đúng
* safety architecture
* triage logic
* temporal reasoning
* exposure reasoning
* disambiguation language layer

Nói thẳng: **90–95% hệ thống symptom checker trên thế giới chưa có mức này.**

Tuy nhiên, nếu mình đứng ở góc nhìn:

> **bác sĩ khám bệnh cực kỳ giỏi, nhiều năm kinh nghiệm**

và **bám sát mục tiêu ban đầu của hệ thống**, thì vẫn còn **4 lớp cực kỳ quan trọng nên bổ sung**.

Các lớp này **không phá kiến trúc v3**, mà chỉ làm hệ thống **giống bác sĩ thật hơn**.

# 1. Thêm “Diagnostic Confidence Engine”

PRD v3 có:

* pattern
* pháp trị
* triage

Nhưng **thiếu một thứ bác sĩ luôn có trong đầu**:

```text
mức độ chắc chắn
```

Trong lâm sàng:

```text
chẩn đoán chắc
chẩn đoán khả năng cao
chẩn đoán cần loại trừ
```

Hệ thống nên có:

```text
diagnostic_confidence
```

Ví dụ:

| confidence | ý nghĩa            |
| ---------- | ------------------ |
| high       | pattern phù hợp rõ |
| moderate   | nhiều khả năng     |
| low        | cần xác nhận       |

Schema đề xuất:

```sql
pattern_confidence:
  supporting_evidence[]
  contradicting_evidence[]
  missing_key_signs[]
```

Ví dụ output:

```
Chứng: Can Dương Vượng
Confidence: Moderate

Dấu hiệu ủng hộ:
✓ đau đầu chiều
✓ cáu gắt
✓ khô miệng

Dấu hiệu chưa rõ:
? chóng mặt
? ù tai
```

Điều này làm hệ thống **giống reasoning bác sĩ thật**.

---

# 2. Thêm “Missing Key Questions Engine”

Bác sĩ giỏi luôn có phản xạ:

```text
còn câu nào quan trọng chưa hỏi?
```

Ví dụ:

```
đau đầu
```

bác sĩ sẽ hỏi:

```
có sốt không
có yếu tay chân không
có nôn không
```

PRD v3 có **pertinent negatives** nhưng chưa có logic:

```text
missing key signs
```

Nên có bảng:

```sql
key_questions_per_pattern
```

Ví dụ:

```
pattern: liver_yang_rising

key_symptoms:
- dizziness
- tinnitus
- irritability
```

Nếu thiếu → engine hỏi thêm 1–2 câu.

Điều này giúp:

```text
giảm false pattern
```

---

# 3. Thêm “Clinical Contradiction Detector”

Trong thực tế, bác sĩ thường gặp:

```text
triệu chứng mâu thuẫn
```

Ví dụ:

```
sợ lạnh
nhưng khát nước nhiều
```

Hoặc:

```
mệt nhiều
nhưng hoạt động bình thường
```

PRD v3 có **Veto rules** nhưng chưa có:

```text
symptom contradiction detection
```

Ví dụ:

```yaml
contradiction_rules:

cold_signs:
  incompatible_with:
    - high_fever
    - intense_thirst

deficiency_signs:
  incompatible_with:
    - sudden_acute_onset
```

Khi phát hiện:

```
flag = clinical_inconsistency
```

→ hỏi lại bệnh nhân.

Điều này **rất giống bác sĩ thật**.

---

# 4. Thêm “Follow-up Safety Loop”

Một bác sĩ giỏi luôn nói:

```
nếu sau X ngày không đỡ → quay lại
```

Hệ thống hiện tại **kết thúc session luôn**.

Nhưng nên thêm:

```text
follow_up_rules
```

Ví dụ:

| pattern           | follow up |
| ----------------- | --------- |
| mild cold pattern | 3 ngày    |
| spleen deficiency | 2 tuần    |
| fatigue unclear   | 7 ngày    |

Output thêm:

```
Nếu sau 3 ngày triệu chứng không cải thiện hoặc tệ hơn,
hãy khám bác sĩ.
```

Điều này cực kỳ quan trọng cho safety.

# 5. Một cải tiến nhỏ nhưng cực kỳ giá trị

### “First symptom vs later symptoms”

Bạn có hierarchy nhưng nên thêm:

```text
symptom chronology
```

Ví dụ:

```
mệt mỏi → mất ngủ
```

khác với:

```
mất ngủ → mệt mỏi
```

Trong YHCT điều này thay đổi:

```
Tâm Thận bất giao
vs
Can khí uất
```

Chỉ cần thêm:

```sql
symptom_order
```
